## 📑 Executive Summary

**The Hunter** is a fully automated, agentic phishing detection system. Moving beyond traditional signature-based filters, this system utilizes a multi-agent artificial intelligence framework (CrewAI) heavily augmented by deep learning (BiLSTM) to autonomously mimic the triage flow of a human Security Operations Center (SOC).

It ingests raw emails, extracts structural features, processes the text through a trained deep learning classifier, and dictates a final verdict (quarantine, log, or pass) while providing a transparent internal chain-of-thought to the end user via a custom dashboard.

## ⚠️ Problem Statement

Modern phishing attacks employ advanced tactics that evade traditional signature-based filters. Current security frameworks often rely on binary, deterministic rules engines (e.g., matching a known malicious IP) or single-layer machine learning classifiers. These systems fail when confronted with zero-day social engineering, ambiguous credential requests, or nuanced text manipulations. 

A static classifier provides a probability score but lacks the semantic reasoning required to investigate *why* an email is malicious. Human analysts possess that semantic reasoning, but scaling manual investigation to thousands of emails daily is impossible.

## 💡 The Solution

We built an architecture that blends the pattern-recognition capabilities of deep learning with the semantic reasoning of Large Language Models (LLMs). 

The system operates on an **Agentic Pipeline** utilizing three distinct LLM-driven personas:
1. **Agent 1 (Ingest Specialist):** Cleans raw email text and extracts signals (URL counts, urgency keywords).
2. **Agent 2 (Risk Analyst):** Interacts natively with the Deep Learning Ensemble to acquire a probability score, decides if the threat requires deeper pattern-matching analysis, and generates a confident risk assessment.
3. **Agent 3 (SOC Orchestrator):** References a persistent SQLite Threat Memory database for repeat offenders, processes the analyst's assessment, and applies a concrete 4-tier defense policy.

### The Deep Learning Ensemble
The core probability scoring is generated by an ensemble model trained on the Alam Dataset (82,000+ emails).
* **Bidirectional LSTM (BiLSTM):** A two-layer stacked network (128 units → 64 units) handling sequential text classification. It uses representation learning via a trainable embedding layer to map structural phishing features. Carries 65% weight.
* **Logistic Regression:** A feature-driven secondary metric targeting static counts (URLs, exclamation marks, etc.) trained via SMOTE oversampling. Carries 35% weight.
